## 1\. O Princípio de Tudo: Objetos Literais

No JavaScript, quase tudo se resume a **Objetos**. Pense em um objeto do mundo real, como um Celular. Ele tem características (cor, marca) e ações que pode executar (ligar, tirar foto).

No código, agrupamos essas informações usando chaves `{}`.

```javascript
const jogador = {
    // PROPRIEDADES (Características - guardam valores)
    nome: "Mário",
    vidas: 3,

    // MÉTODOS (Ações - guardam funções)
    pular: function() {
        console.log("Yahoo! O Mário pulou!");
    }
};

// Lendo uma propriedade (usa-se o ponto)
console.log(jogador.nome); // Imprime: Mário

// Executando um método (PRECISA dos parênteses para a ação ocorrer)
jogador.pular(); // Imprime: Yahoo! O Mário pulou!
```

> **Regra de Ouro:** Se tem `()` no final, é uma ação (método). Se não tem, é uma característica (propriedade).

## 2\. O Problema da Repetição

Objetos literais são ótimos para coisas únicas. Mas e se você estiver criando um sistema para uma concessionária e precisar cadastrar centenas de carros?

```javascript
const carro1 = { marca: "Fiat", modelo: "Uno", ligado: false };
const carro2 = { marca: "Ford", modelo: "Ka", ligado: false };
// Imagina digitar isso 1000 vezes... E se esquecermos uma vírgula?
```

Fazer isso na mão gera código frágil e repetitivo. Precisamos de uma "fábrica" de carros.

## 3\. A Solução: Classes (Fábricas de Objetos)

Uma **Classe** é como a **planta baixa** de uma casa. Você não pode morar na planta, ela é apenas o *molde*. As casas reais que você constrói a partir desse molde são os **Objetos** (ou Instâncias).

Para criar uma classe, usamos a função especial `constructor`. Ele é o motor que roda automaticamente na hora em que pedimos para fabricar um objeto novo.

```javascript
// Convenção: Nomes de Classes sempre começam com Letra Maiúscula
class Carro {
    // O constructor recebe os "ingredientes"
    constructor(marcaEscolhida, modeloEscolhido) {
        // 'this' significa 'ESTE objeto específico que estou fabricando agora'
        this.marca = marcaEscolhida;
        this.modelo = modeloEscolhido;
        this.ligado = false; // Valor padrão para todos os carros fabricados
    }
}

// Fabricando (instanciando) os objetos usando a palavra 'new'
const meuCarro = new Carro("Honda", "Civic");
const carroDoChefe = new Carro("BMW", "X5");

console.log(meuCarro.marca); // "Honda"
console.log(carroDoChefe.modelo); // "X5"
```

## 4\. Dando Vida: Adicionando Métodos

As classes ficam realmente poderosas quando ensinamos a elas comportamentos. Escrevemos a ação uma única vez na Classe, e todos os objetos fabricados ganham esse poder.

```javascript
class Usuario {
    constructor(nome, idade) {
        this.nome = nome;
        this.idade = idade;
    }

    // Criando um método dentro da classe (não precisa da palavra 'function')
    seApresentar() {
        console.log(`Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`);
    }
}

const user1 = new Usuario("Ana", 25);
const user2 = new Usuario("Carlos", 30);

// Ambos sabem se apresentar usando o mesmo molde!
user1.seApresentar();
user2.seApresentar();
```

## 5\. Herança: Passando o DNA Adiante (`extends`)

E se quisermos criar um `UsuarioVIP` que faz tudo o que o usuário comum faz, mas tem privilégios extras? Não precisamos copiar e colar código. Usamos a **Herança**.

```javascript
// UsuarioVIP 'estende' (herda) a base de Usuario
class UsuarioVIP extends Usuario {
    constructor(nome, idade, nivelBonus) {
        // O 'super' chama o construtor da classe pai (Usuario) para fazer o trabalho pesado
        super(nome, idade); 
        this.nivelBonus = nivelBonus;
    }

    // Novo método exclusivo do VIP
    acessarSalaVip() {
        console.log(`${this.nome} entrou na área VIP com bônus nível ${this.nivelBonus}!`);
    }
}

const chefao = new UsuarioVIP("Roberto", 45, 5);

chefao.seApresentar();   // Herdado do Usuario comum!
chefao.acessarSalaVip(); // Método exclusivo do VIP.
```

## 6\. ⚔️ Laboratório Prático: RPG no Console

**O Desafio:**
Vocês vão programar a base de um jogo de RPG puramente em texto rodando no console do navegador (F12).

**Requisitos:**

1.  Crie uma classe base chamada `Personagem` com as propriedades: `nome`, `vida`, e `poder`.
2.  A classe deve ter um método `atacar()`, que exibe: `[NOME] atacou com força de [PODER]!`.
3.  Crie uma classe `Mago` que herda de `Personagem`. O construtor do `Mago` deve receber um atributo extra: `mana`.
4.  O `Mago` deve ter um método `curar()`, que aumenta sua própria vida em 20 pontos e gasta 10 de mana (só pode curar se tiver mana suficiente).
5.  Instancie um Guerreiro (Personagem comum) e um Mago. Teste os ataques e a cura no console\!
