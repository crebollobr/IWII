## 📅 Dominando Formulários e Elementos Dinâmicos

Na segunda noite, vamos focar em entrada de dados. Como lemos o que o usuário digita e como criamos novos elementos no HTML que não existiam antes?

### 📖 Teoria: Valores e Criação de Elementos
* **Lendo Inputs:** Diferente de uma `div` onde mudamos a cor, em um `<input>` nós queremos o **texto** que está lá dentro. Usamos `elemento.value`.
* **Impedindo o Refresh:** Formulários HTML têm a mania de recarregar a página ao enviar. Bloqueamos isso com o `evento.preventDefault()`.
* **Criando do Zero:** 1. `document.createElement('option')`: Nasce um novo elemento na memória do JS.
  2. `elemento.textContent = "Novo Item"`: Damos um texto a ele.
  3. `caixaMae.appendChild(elemento)`: Colamos ele na tela, dentro da caixa desejada.

### 💻 Laboratório Prático 02: O Select Box Dinâmico

Esqueça as páginas estáticas. Vamos criar um campo onde o usuário digita um item (ex: "Maçã"), clica em "Adicionar", e esse item entra magicamente em uma lista de seleção (Select Box)!

1. **A Estrutura HTML:**
   ```html
   <h2>Lista de Compras Dinâmica</h2>
   
   <form id="meu-formulario">
     <input type="text" id="novo-item" placeholder="Digite um item..." required>
     <button type="submit">Adicionar à Lista</button>
   </form>

   <br>

   <select id="caixa-selecao">
     <option value="">Selecione um item...</option>
     </select>
   ```

2. **O Motor em JavaScript:**
   ```javascript
   // 1. Mapeando os elementos
   const formulario = document.getElementById('meu-formulario');
   const inputItem = document.getElementById('novo-item');
   const selectBox = document.getElementById('caixa-selecao');

   // 2. Escutando o evento de 'submit' (envio) do formulário
   formulario.addEventListener('submit', function(evento) {
       // PÁRA TUDO! Não recarregue a página!
       evento.preventDefault();

       // 3. Pega o texto que o aluno digitou
       const textoDigitado = inputItem.value;

       // 4. Cria um novo elemento de opção <option>
       const novaOpcao = document.createElement('option');
       
       // 5. Configura a nova opção
       novaOpcao.textContent = textoDigitado; // O texto que aparece
       novaOpcao.value = textoDigitado;       // O valor no sistema

       // 6. Adiciona a nova opção dentro do nosso <select>
       selectBox.appendChild(novaOpcao);

       // 7. Limpa a caixa de texto para o próximo item e foca nela
       inputItem.value = "";
       inputItem.focus();
   });
   ```

Mil desculpas! O formato de blocos de código que eu usei na mensagem anterior acabou "quebrando" a exibição na sua tela e ocultando o texto. 

Vamos direto ao ponto, sem enrolação na formatação, para garantir que você consiga ler e copiar tudo. Aqui está o **Guia JavaScript para Inputs e Formulários**, incluindo as suas ideias e mais algumas bem legais (como botões de rádio, slider e seletor de cores):

### 🔤 1. Campo de Texto (`<input type="text">`)
Serve para textos, senhas (`password`), e-mails, etc. A regra principal é usar o `.value`.

**HTML:**
```html
<input type="text" id="meu-texto">
```

**JavaScript:**
```javascript
const campoTexto = document.getElementById('meu-texto');

// 1. Preencher com um texto (Escrever)
campoTexto.value = "Olá, alunos!";

// 2. Ler o que o usuário digitou
let textoDigitado = campoTexto.value;

// 3. Desabilitar (Fica cinza e o usuário não consegue clicar)
campoTexto.disabled = true; 

// 4. Somente Leitura (Consegue copiar, mas não consegue alterar)
campoTexto.readOnly = true;
```

### 📋 2. Caixa de Seleção (`<select>`)
O famoso "Dropdown". Trabalhar com ele envolve a caixa principal e os itens (`<option>`).

**HTML:**
```html
<select id="meu-select">
  <option value="br">Brasil</option>
  <option value="us">Estados Unidos</option>
</select>
```

**JavaScript:**
```javascript
const caixaSelect = document.getElementById('meu-select');

// 1. Alert com o valor do item que está selecionado agora
alert("O valor selecionado é: " + caixaSelect.value);

// 2. Adicionar um novo elemento na lista (Modo direto)
// new Option("Texto que o usuário lê", "valor-oculto-do-sistema")
const novaOpcao = new Option("Japão", "jp");
caixaSelect.add(novaOpcao);

// 3. Remover um elemento (Você informa a posição dele, começando do zero)
caixaSelect.remove(1); // Vai remover a posição 1 (Estados Unidos)
```

### ☑️ 3. Caixa de Marcar (`<input type="checkbox">`)
O Checkbox não usa o `.value` para saber se está marcado, ele usa a propriedade `.checked` (que é sempre `true` ou `false`).

**HTML:**
```html
<input type="checkbox" id="aceito-termos"> Li e aceito os termos
```

**JavaScript:**
```javascript
const checkboxTermos = document.getElementById('aceito-termos');

// 1. Selecionar automaticamente pelo código
checkboxTermos.checked = true;

// 2. Desmarcar
checkboxTermos.checked = false;

// 3. Verificar se está selecionado (Ótimo para usar com "If")
if (checkboxTermos.checked === true) {
    alert("Você aceitou os termos!");
} else {
    alert("Por favor, marque a caixa de termos.");
}
```

### 🔘 4. Botões de Opção (Radio Buttons) 
Diferente do Checkbox, no Radio Button o usuário só pode escolher **uma** opção dentro de um grupo.

**HTML:**
```html
<input type="radio" name="pagamento" value="dinheiro"> Dinheiro
<input type="radio" name="pagamento" value="cartao"> Cartão
```

**JavaScript:**
```javascript
// O truque aqui é usar o querySelector para achar especificamente o que está :checked
const opcaoEscolhida = document.querySelector('input[name="pagamento"]:checked');

if (opcaoEscolhida) {
    alert("Forma de pagamento: " + opcaoEscolhida.value);
} else {
    alert("Escolha uma forma de pagamento!");
}
```

### 🎚️ 5. Slider de Arrastar (`<input type="range">`) [Ideia Extra 2]
Excelente para a aula! Dá para fazer calculadoras ou controles visuais de tamanho.

**HTML:**
```html
<input type="range" id="volume" min="0" max="100" value="50">
```

**JavaScript:**
```javascript
const controleVolume = document.getElementById('volume');

// Lê o número exato onde o usuário parou a "bolinha"
alert("O nível escolhido foi: " + controleVolume.value);
```

### 🎨 6. Seletor de Cores (`<input type="color">`) 
Abre a paleta de cores do sistema operacional. Os alunos piram com esse porque dá para mudar a cor do site em tempo real.

**HTML:**
```html
<input type="color" id="cor-fundo"> Escolha a cor
```

**JavaScript:**
```javascript
const seletorCor = document.getElementById('cor-fundo');

// Retorna a cor em Hexadecimal (ex: #ff0000 para vermelho)
let corEscolhida = seletorCor.value;

// Brincadeira para a aula: Aplicar a cor escolhida no fundo da página!
document.body.style.backgroundColor = corEscolhida;
```

### 🛡️ Validação de Formulário: Nome, Data e E-mail

Neste exemplo, vamos criar uma `div` específica para mostrar os erros em vermelho na tela, o que é muito mais amigável do que ficar jogando vários `alerts` na cara do usuário.

#### 1. O HTML (A Estrutura)
O tipo do input ajuda muito (ex: `type="date"` já abre o calendário, `type="email"` já ajuda no celular), mas o JavaScript é quem vai fazer a conferência final.

```html
<form id="form-usuario">
  <label for="nome">Nome Completo:</label><br>
  <input type="text" id="nome" placeholder="Digite seu nome"><br><br>

  <label for="data-nasc">Data de Nascimento:</label><br>
  <input type="date" id="data-nasc"><br><br>

  <label for="email">E-mail:</label><br>
  <input type="email" id="email" placeholder="usuario@email.com"><br><br>

  <button type="submit">Cadastrar</button>
</form>

<div id="mensagem-erro" style="color: red; margin-top: 15px; font-weight: bold;"></div>
```

#### 2. O JavaScript (O Juiz)
Vamos pegar os valores (`.value`) e testar um por um. Se acharmos um erro, guardamos em uma lista.

```javascript
// 1. Mapear os elementos do HTML
const formulario = document.getElementById('form-usuario');
const inputNome = document.getElementById('nome');
const inputDataNasc = document.getElementById('data-nasc');
const inputEmail = document.getElementById('email');
const divErro = document.getElementById('mensagem-erro');

// 2. Escutar o envio do formulário
formulario.addEventListener('submit', function(evento) {
    // Impede a página de recarregar (CRUCIAL!)
    evento.preventDefault();
    
    // Limpa os erros anteriores da tela sempre que tenta enviar de novo
    divErro.innerHTML = "";
    
    // Criamos uma lista vazia para guardar os problemas que encontrarmos
    let erros = [];

    // --- VALIDAÇÃO DO NOME ---
    // O .trim() remove espaços em branco inúteis antes e depois do texto
    if (inputNome.value.trim() === "") {
        erros.push("O campo Nome não pode estar vazio.");
    } else if (inputNome.value.length < 3) {
        erros.push("O Nome deve ter pelo menos 3 letras.");
    }

    // --- VALIDAÇÃO DA DATA DE NASCIMENTO ---
    if (inputDataNasc.value === "") {
        erros.push("Por favor, preencha a Data de Nascimento.");
    } else {
        // Lógica Extra (Desafio): Verificar se o usuário não colocou uma data no futuro!
        let dataDigitada = new Date(inputDataNasc.value);
        let dataDeHoje = new Date();
        
        if (dataDigitada > dataDeHoje) {
            erros.push("Você não pode ter nascido no futuro!");
        }
    }

    // --- VALIDAÇÃO DO E-MAIL ---
    if (inputEmail.value === "") {
        erros.push("O campo E-mail é obrigatório.");
    } else if (!inputEmail.value.includes('@') || !inputEmail.value.includes('.')) {
        // includes() verifica se um caractere existe dentro do texto
        erros.push("Digite um E-mail válido (deve conter '@' e '.').");
    }

    // --- DECISÃO FINAL: PASSA OU NÃO PASSA? ---
    if (erros.length > 0) {
        // Se a lista de erros for maior que zero, mostramos tudo na div
        // O .join("<br>") junta os textos da lista quebrando a linha
        divErro.innerHTML = erros.join("<br>");
    } else {
        // Se a lista estiver vazia, está tudo perfeito!
        alert("Cadastro realizado com sucesso!");
        
        // Limpa todos os campos do formulário de uma vez
        formulario.reset(); 
    }
});
```
